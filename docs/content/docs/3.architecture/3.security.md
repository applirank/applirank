# Security

Reqcore implements defense-in-depth security across authentication, authorization, data access, file handling, and network boundaries.

## Authentication

| Layer | Implementation |
|-------|---------------|
| **Session management** | HTTP-only cookies via Better Auth |
| **Password hashing** | Argon2id (Better Auth default) |
| **CSRF protection** | Origin header validation |
| **Auth guard** | `requireAuth(event)` on every protected endpoint |

## Authorization (Multi-Tenancy)

The #1 security invariant in Reqcore:

::callout{icon="i-lucide-shield-alert" color="error"}
**The organization ID is ALWAYS extracted from the authenticated session, NEVER from user input.** Every database query is scoped by `organizationId`. There is no way for a user to access another organization's data.
::

```ts
// ✅ Correct: orgId comes from session
const { orgId } = await requireAuth(event)
const jobs = await db.select().from(schema.job)
  .where(eq(schema.job.organizationId, orgId))

// ❌ Never: orgId from request
const { orgId } = getQuery(event) // NEVER DO THIS
```

## Input Validation

All user input is validated with Zod v4 schemas:

```ts
const body = await readValidatedBody(event, createJobSchema.parse)
```

Validation schemas are defined in `server/utils/schemas/` and shared between client and server where applicable.

## Rate Limiting

Public endpoints are protected by an IP-based sliding window rate limiter:

```ts
const limiter = createRateLimiter({ max: 10, window: 60 })
```

| Endpoint Type | Limit |
|--------------|-------|
| Public API (job board) | Standard limits |
| Authentication | Stricter limits |
| Write operations | Stricter limits |

## Document Security

| Measure | Details |
|---------|---------|
| **Server-proxied access** | All file downloads and previews stream through the authenticated Nitro server |
| **No presigned URLs** | S3 URLs are never exposed to clients, preventing URL sharing |
| **Private bucket policy** | MinIO public policies are deleted on every startup |
| **MIME validation** | Uses `file-type` package to validate magic bytes, not just headers |
| **Filename sanitization** | `sanitizeFilename()` prevents path traversal, XSS, and filesystem exploits |
| **Per-candidate limits** | Maximum 20 documents per candidate on public endpoints |
| **Preview restrictions** | Only PDF files can be previewed; DOC/DOCX must be downloaded |
| **Cache headers** | `Cache-Control: private, no-store` on all document responses |
| **Frame restrictions** | `X-Frame-Options: DENY` globally, `SAMEORIGIN` for PDF preview only |

## Security Headers

Applied globally via Nitro route rules:

| Header | Value |
|--------|-------|
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `DENY` |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `X-XSS-Protection` | `1; mode=block` |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` |

## Environment Security

- All environment variables are validated at startup with Zod
- Secrets are never exposed to the client
- Server-only runtime config is used for sensitive values
- `.env` files are git-ignored

## Public vs. Authenticated Routes

| Route Pattern | Auth Required | Description |
|---------------|:---:|-------------|
| `/api/public/**` | ❌ | Public job board data (open jobs only) |
| `/api/auth/**` | ❌ | Authentication endpoints |
| `/api/jobs/**` | ✅ | Authenticated job management |
| `/api/candidates/**` | ✅ | Authenticated candidate management |
| `/api/documents/**` | ✅ | Authenticated document access |

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email security concerns to the maintainers
3. See [SECURITY.md](https://github.com/reqcore-inc/reqcore/blob/main/SECURITY.md) for the full security policy
