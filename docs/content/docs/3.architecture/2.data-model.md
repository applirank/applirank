# Data Model

Reqcore's data model is designed around **multi-tenant organizations** where every piece of data belongs to exactly one organization.

## Entity Relationship

```
organization (Better Auth)
├── job (draft → open → closed → archived)
│   ├── question (custom application questions)
│   └── application (new → screening → interview → offer → hired/rejected)
│       ├── candidate
│       │   └── document (resume, cover_letter)
│       └── answer (responses to custom questions)
└── member (user ↔ organization with role)
```

## Core Tables

### `job`

Represents a job posting within an organization.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `organizationId` | UUID | Foreign key to organization |
| `title` | VARCHAR | Job title |
| `description` | TEXT | Job description (rich text) |
| `location` | VARCHAR | Job location |
| `employmentType` | ENUM | full-time, part-time, contract, internship |
| `remoteStatus` | ENUM | on-site, hybrid, remote |
| `salaryMin` | INTEGER | Minimum salary (optional) |
| `salaryMax` | INTEGER | Maximum salary (optional) |
| `salaryCurrency` | VARCHAR | Currency code (e.g., USD, EUR) |
| `status` | ENUM | draft, open, closed, archived |
| `slug` | VARCHAR | URL-friendly identifier for public pages |
| `createdAt` | TIMESTAMP | Creation timestamp |
| `updatedAt` | TIMESTAMP | Last update timestamp |

#### Status Workflow

```
draft → open → closed → archived
```

- **Draft** — Job is being prepared, not visible publicly.
- **Open** — Job is published on the public job board and accepting applications.
- **Closed** — Job is no longer accepting applications.
- **Archived** — Job is hidden from active listings.

### `candidate`

Represents a person who has applied or been added to the system.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `organizationId` | UUID | Foreign key to organization |
| `firstName` | VARCHAR | First name |
| `lastName` | VARCHAR | Last name |
| `email` | VARCHAR | Email address (unique per org) |
| `phone` | VARCHAR | Phone number (optional) |
| `createdAt` | TIMESTAMP | Creation timestamp |

Candidates are **deduplicated** within each organization by email:

```
uniqueIndex(organizationId, email)
```

### `application`

Links a candidate to a job with a status representing their position in the pipeline.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `jobId` | UUID | Foreign key to job |
| `candidateId` | UUID | Foreign key to candidate |
| `organizationId` | UUID | Foreign key to organization |
| `status` | ENUM | new, screening, interview, offer, hired, rejected |
| `createdAt` | TIMESTAMP | Application timestamp |

#### Pipeline Stages

```
new → screening → interview → offer → hired
                                    → rejected
```

### `document`

Stores metadata for uploaded files (resumes, cover letters). The actual file binary is in S3.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `candidateId` | UUID | Foreign key to candidate |
| `organizationId` | UUID | Foreign key to organization |
| `type` | ENUM | resume, cover_letter |
| `originalName` | VARCHAR | Sanitized original filename |
| `mimeType` | VARCHAR | File MIME type |
| `size` | INTEGER | File size in bytes |
| `storageKey` | VARCHAR | S3 object key (never exposed to clients) |
| `createdAt` | TIMESTAMP | Upload timestamp |

### `question`

Custom questions added to a job's application form.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `jobId` | UUID | Foreign key to job |
| `organizationId` | UUID | Foreign key to organization |
| `label` | VARCHAR | Question text |
| `type` | ENUM | text, textarea, select, file |
| `options` | JSONB | Options for select-type questions |
| `required` | BOOLEAN | Whether the question is mandatory |
| `order` | INTEGER | Display order |

### `answer`

Responses to custom questions submitted with an application.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `applicationId` | UUID | Foreign key to application |
| `questionId` | UUID | Foreign key to question |
| `value` | TEXT | Answer text |

## Auth Tables

Better Auth manages its own tables. **Do not modify these directly**:

- `user` — User accounts
- `session` — Active sessions
- `account` — OAuth provider accounts
- `organization` — Organizations
- `member` — User-organization memberships with roles
