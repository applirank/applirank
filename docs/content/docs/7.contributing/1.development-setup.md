# Contributing

Reqcore is an open-source project and contributions are welcome. This guide covers the development workflow, coding conventions, and submission process.

## Development Setup

### Prerequisites

- **Node.js** 18.20+ (LTS 20+ recommended)
- **Docker** with Docker Compose v2
- **Git**

### Clone and Install

```bash
git clone https://github.com/reqcore-inc/reqcore.git
cd reqcore
npm install
```

### Start Development Services

```bash
# Start Postgres + MinIO
docker compose up db minio -d

# Generate secrets
./setup.sh

# Start Nuxt dev server
npm run dev
```

The app is available at [http://localhost:3000](http://localhost:3000).

### Seed Demo Data

```bash
npm run db:seed
```

Sign in with `demo@reqcore.com` / `demo1234`.

## Development Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start development server with HMR |
| `npm run build` | Build for production |
| `npm run preview` | Preview production build locally |
| `npm run db:generate` | Generate migrations from schema changes |
| `npm run db:migrate` | Apply migrations |
| `npm run db:push` | Push schema directly (dev only) |
| `npm run db:studio` | Open Drizzle Studio |
| `npm run db:seed` | Seed demo data |
| `npm run test:e2e` | Run Playwright E2E tests |

## Project Conventions

### Directory Structure

- Frontend code in `app/`
- Server code in `server/`
- Shared types in `shared/`
- Tests in `e2e/`

### Coding Style

- **TypeScript** everywhere
- **Zod v4** for validation schemas
- **Drizzle ORM** for database queries
- **Auto-imports** — don't manually import Nuxt/Vue composables
- **`createError()`** for server errors (never return error objects)

### Commit Messages

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add interview scheduling
fix: prevent duplicate candidate on concurrent apply
docs: update self-hosting guide
chore: update dependencies
```

### DCO Sign-Off

All contributions require a [Developer Certificate of Origin](https://developercertificate.org/) sign-off:

```bash
git commit -s -m "feat: add interview scheduling"
```

## Submitting Changes

1. **Fork** the repository
2. **Create a branch** from `main`: `git checkout -b feat/my-feature`
3. **Make your changes** with tests where applicable
4. **Run tests**: `npm run test:e2e`
5. **Commit** with DCO sign-off
6. **Push** and open a **Pull Request**

### PR Guidelines

- Keep PRs focused on a single concern
- Include a clear description of what changed and why
- Reference any related issues
- Ensure all CI checks pass
