# Docker

Reqcore uses Docker for both local development and production deployment.

## Docker Compose (Development)

The `docker-compose.yml` file defines three services:

| Service | Image | Port | Purpose |
|---------|-------|------|---------|
| `app` | Built from `Dockerfile` | 3000 | Nuxt application |
| `db` | `postgres:16-alpine` | 5432 | PostgreSQL database |
| `minio` | `minio/minio` | 9000, 9001 | S3-compatible storage |

Optional service (requires `--profile tools`):

| Service | Image | Port | Purpose |
|---------|-------|------|---------|
| `adminer` | `adminer` | 8080 | Database browser |

### Starting Development

```bash
# Start all services
docker compose up

# Start with database browser
docker compose --profile tools up

# Start in background
docker compose up -d

# View logs
docker compose logs -f app
```

### Stopping

```bash
# Stop and keep data
docker compose down

# Stop and delete ALL data (irreversible)
docker compose down -v
```

## Dockerfile (Production)

The `Dockerfile` uses a multi-stage build for minimal production images:

### Stage 1: Dependencies

Installs Node.js dependencies from `package.json`.

### Stage 2: Build

Runs `nuxt build` to generate the production server in `.output/`.

### Stage 3: Runtime

Copies only the built output and runs with `node .output/server/index.mjs`. The final image is minimal — no source code, no dev dependencies.

## Building Manually

```bash
# Build the Docker image
docker build -t reqcore .

# Run with environment variables
docker run -p 3000:3000 \
  --env-file .env \
  reqcore
```

## Docker Compose Override

For local customization, create a `docker-compose.override.yml`:

```yaml
services:
  app:
    environment:
      - NUXT_PUBLIC_SITE_URL=http://localhost:3000
    volumes:
      - .:/app
      - /app/node_modules
```

## Health Checks

The default Docker Compose includes health checks for the database and MinIO services. The app service waits for these to be healthy before starting.
