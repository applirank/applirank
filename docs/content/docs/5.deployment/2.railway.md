# Railway Deployment

[Railway](https://railway.app) offers managed deployment with automatic builds from GitHub, PostgreSQL, and S3-compatible storage.

## Architecture

| Component | Railway Service |
|-----------|----------------|
| **App** | Nuxt service (auto-built from GitHub via Nixpacks) |
| **Database** | Railway PostgreSQL (managed, automatic backups) |
| **Storage** | Railway Storage Bucket (S3-compatible) |

## Setup

### 1. Create a Railway Project

1. Log in to [Railway](https://railway.app)
2. Create a new project
3. Connect your GitHub repository (`reqcore-inc/reqcore`)

### 2. Add Services

Add the following services to your project:

- **PostgreSQL** — Click "Add Service" → PostgreSQL
- **Storage Bucket** — Click "Add Service" → Object Storage

### 3. Configure Environment Variables

In your Nuxt service settings, add these variables:

| Variable | Value |
|----------|-------|
| `DATABASE_URL` | `${{Postgres.DATABASE_URL}}` |
| `S3_ENDPOINT` | `${{Bucket.ENDPOINT}}` |
| `S3_ACCESS_KEY` | `${{Bucket.ACCESS_KEY_ID}}` |
| `S3_SECRET_KEY` | `${{Bucket.SECRET_ACCESS_KEY}}` |
| `S3_BUCKET` | `${{Bucket.BUCKET}}` |
| `S3_REGION` | `${{Bucket.REGION}}` |
| `S3_FORCE_PATH_STYLE` | `false` |
| `BETTER_AUTH_SECRET` | Generate with `openssl rand -base64 32` |
| `BETTER_AUTH_URL` | `https://${{RAILWAY_PUBLIC_DOMAIN}}` |

### 4. Deploy

Push to `main` — Railway auto-builds and deploys:

```bash
git push origin main
```

Build command: `npm run build` (auto-detected from `package.json`)
Start command: `node .output/server/index.mjs`

## Custom Domain

1. In Railway, go to your service's Settings → Domains
2. Add your custom domain (e.g., `ats.yourcompany.com`)
3. In your DNS provider, create a CNAME record pointing to the Railway domain
4. Update `BETTER_AUTH_URL` to your custom domain

## Preview Environments

Railway can create preview environments for pull requests:

- Each PR gets its own deployment with a unique URL
- Set `BETTER_AUTH_URL` to `https://${{RAILWAY_PUBLIC_DOMAIN}}` in preview environment variables

## Cloudflare Integration

For DDoS protection and edge caching:

1. Set up Cloudflare DNS for your domain
2. Create a CNAME record pointing to your Railway domain
3. Enable Cloudflare proxy (orange cloud)
4. Configure `NUXT_PUBLIC_SITE_URL` to your Cloudflare domain
