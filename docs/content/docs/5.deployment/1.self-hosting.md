# Self-Hosting

Deploy Reqcore on your own infrastructure with full control over your data. This guide covers a production setup with Docker Compose, Caddy reverse proxy, and Cloudflare DNS.

## Reference Architecture

| Component | Role | Cost |
|-----------|------|------|
| **VPS** | Hetzner CX23 (2 vCPU, 4 GB RAM, Ubuntu 24.04) | ~€5/mo |
| **Caddy** | Reverse proxy with automatic HTTPS | Free |
| **Cloudflare** | DNS, DDoS protection, edge SSL | Free |
| **Docker Compose** | Runs Postgres + MinIO (localhost only) | Free |
| **systemd** | Process management with auto-restart | Built-in |

Total cost: **~€5/month** for a small to medium hiring team.

## Prerequisites

- A Linux VPS (Ubuntu 22.04+ recommended)
- A domain name (e.g., `ats.yourcompany.com`)
- SSH access to the server
- Docker & Docker Compose installed

## Step 1 — Install Docker

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

Log out and back in for the group change to take effect.

## Step 2 — Install Caddy

```bash
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy
```

## Step 3 — Configure DNS

In Cloudflare (or your DNS provider), create an A record:

| Type | Name | Content | Proxy |
|------|------|---------|-------|
| A | `ats` | Your server's IP | Proxied (orange cloud) |

## Step 4 — Clone and Configure

```bash
git clone https://github.com/reqcore-inc/reqcore.git /opt/reqcore
cd /opt/reqcore
./setup.sh
```

Edit `.env` and update for production:

```ini
BETTER_AUTH_URL=https://ats.yourcompany.com
NUXT_PUBLIC_SITE_URL=https://ats.yourcompany.com
```

## Step 5 — Configure Caddy

Create `/etc/caddy/Caddyfile`:

```caddyfile
ats.yourcompany.com {
    reverse_proxy localhost:3000
}
```

Restart Caddy:

```bash
sudo systemctl restart caddy
```

Caddy automatically obtains and renews SSL certificates.

## Step 6 — Start Reqcore

```bash
cd /opt/reqcore
docker compose up -d
```

Verify it's running:

```bash
docker compose logs app --tail 20
# Look for: Listening on http://[::]:3000
```

Visit `https://ats.yourcompany.com` — you should see the Reqcore landing page.

## Automated Deployments

Create a deploy script at `~/deploy.sh`:

```bash
#!/bin/bash
set -e
cd /opt/reqcore
git pull origin main
docker compose up --build -d
echo "✅ Deployed $(git rev-parse --short HEAD)"
```

Deploy from your local machine:

```bash
ssh deploy@your-server '~/deploy.sh'
```

## Backups

### Database

```bash
# Backup
docker compose exec db pg_dump -U reqcore reqcore > backup_$(date +%Y%m%d).sql

# Restore
docker compose exec -i db psql -U reqcore reqcore < backup_20260101.sql
```

### Documents (MinIO)

```bash
# Install MinIO client
curl -O https://dl.min.io/client/mc/release/linux-amd64/mc
chmod +x mc
sudo mv mc /usr/local/bin/

# Configure
mc alias set local http://localhost:9000 minioadmin minioadmin

# Backup
mc mirror local/reqcore /backups/reqcore-docs/
```

## Monitoring

### Health Check

The application serves an SSR page at `/`. A simple HTTP check confirms it's running:

```bash
curl -s -o /dev/null -w "%{http_code}" https://ats.yourcompany.com
# Expected: 200
```

### Logs

```bash
# App logs
docker compose logs app --tail 50 -f

# Database logs
docker compose logs db --tail 50 -f

# MinIO logs
docker compose logs minio --tail 50 -f
```

### systemd Service (Optional)

Create `/etc/systemd/system/reqcore.service` for auto-restart:

```ini
[Unit]
Description=Reqcore ATS
After=docker.service
Requires=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/opt/reqcore
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable reqcore
sudo systemctl start reqcore
```
