# API Overview

Reqcore exposes a REST API via Nitro server routes. All endpoints are under `/api/`.

## Authentication

Most endpoints require authentication via session cookies. Include the session cookie with every request.

For server-side rendering, forward cookies:

```ts
const { data } = await useFetch('/api/jobs', {
  headers: useRequestHeaders(['cookie']),
})
```

### Auth Guard

Protected endpoints use `requireAuth(event)` which:

1. Validates the session cookie
2. Returns the session and active `organizationId`
3. Throws `401` if not authenticated
4. Throws `403` if no active organization

## Error Handling

Errors are returned as JSON with appropriate HTTP status codes:

```json
{
  "statusCode": 404,
  "message": "Job not found"
}
```

| Status Code | Meaning |
|-------------|---------|
| `400` | Bad request (validation error) |
| `401` | Not authenticated |
| `403` | Forbidden (no active organization or insufficient permissions) |
| `404` | Resource not found |
| `429` | Rate limit exceeded |
| `500` | Internal server error |

## Rate Limiting

All `/api` endpoints are rate-limited per IP address using a sliding window algorithm:

| Endpoint Type | Limit |
|--------------|-------|
| General API | Standard |
| Authentication | Stricter |
| Write operations | Stricter |
| Public apply | Stricter |

When rate-limited, the API returns `429 Too Many Requests`.

## Response Format

All successful responses return JSON. Lists return arrays, single resources return objects:

```json
// List
[
  { "id": "...", "title": "Senior Engineer", ... },
  { "id": "...", "title": "Product Manager", ... }
]

// Single
{ "id": "...", "title": "Senior Engineer", ... }
```

## Endpoints

### Jobs (Authenticated)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/jobs` | List all jobs in the organization |
| `POST` | `/api/jobs` | Create a new job |
| `GET` | `/api/jobs/:id` | Get a specific job |
| `PATCH` | `/api/jobs/:id` | Update a job |
| `DELETE` | `/api/jobs/:id` | Delete a job |

### Job Questions (Authenticated)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/jobs/:id/questions` | List questions for a job |
| `POST` | `/api/jobs/:id/questions` | Add a question to a job |
| `PATCH` | `/api/jobs/:id/questions/:qid` | Update a question |
| `DELETE` | `/api/jobs/:id/questions/:qid` | Delete a question |

### Candidates (Authenticated)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/candidates` | List all candidates in the organization |
| `GET` | `/api/candidates/:id` | Get a specific candidate |

### Applications (Authenticated)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/jobs/:id/applications` | List applications for a job |
| `PATCH` | `/api/applications/:id` | Update application status |

### Documents (Authenticated)

| Method | Path | Description |
|--------|------|-------------|
| `DELETE` | `/api/documents/:id` | Delete a document |
| `GET` | `/api/documents/:id/download` | Download a document |
| `GET` | `/api/documents/:id/preview` | Preview a PDF document |

### Public (No Authentication)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/public/jobs` | List open jobs |
| `GET` | `/api/public/jobs/:slug` | Get a specific open job by slug |
| `POST` | `/api/public/jobs/:slug/apply` | Submit an application |

### Authentication

| Method | Path | Description |
|--------|------|-------------|
| `ALL` | `/api/auth/*` | Better Auth catch-all (sign-up, sign-in, session, etc.) |

## Input Validation

All write endpoints validate input using Zod v4 schemas:

```ts
// Server-side validation
const body = await readValidatedBody(event, createJobSchema.parse)
```

Validation schemas are defined in `server/utils/schemas/` and enforce:

- Required fields
- String length limits
- Email format
- Enum values
- File type restrictions
