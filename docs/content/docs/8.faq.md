# Frequently Asked Questions

Common questions about Reqcore, answered concisely.

## General

### What is Reqcore?

Reqcore is an open-source applicant tracking system (ATS) built with Nuxt 4. It lets companies manage their entire recruitment pipeline — from posting jobs to hiring candidates — without vendor lock-in or per-seat pricing. You self-host it on your own infrastructure and own all your data.

### Is Reqcore free?

Yes. Reqcore is licensed under the AGPL-3.0 license and is completely free to use, modify, and deploy. The only costs are your hosting infrastructure (a small VPS starts at ~€5/month).

### What makes Reqcore different from other ATS platforms?

Three things: **data ownership** (you own the database and file storage), **transparent AI** (when AI features ship, every decision will show visible matching logic), and **no per-seat pricing** (your team can grow without increasing your software bill).

### Is Reqcore production-ready?

Reqcore is in active development. The MVP features (jobs, candidates, pipeline, documents, public job board, dashboard) are complete and functional. AI-powered features like resume parsing and candidate ranking are on the roadmap.

## Self-Hosting

### What do I need to self-host Reqcore?

A Linux VPS with Docker installed. The reference setup uses a Hetzner CX23 (2 vCPU, 4 GB RAM, ~€5/month), but any VPS with Docker support works. See the [self-hosting guide](/docs/deployment/self-hosting).

### Can I run Reqcore on Windows?

Reqcore is designed for Linux servers. For local development on Windows, use WSL2 (Windows Subsystem for Linux) or Git Bash. The Docker setup works on all platforms.

### How do I backup my data?

Database: Use `pg_dump` via Docker Compose. Documents: Use the MinIO client (`mc`) to mirror the bucket. See the [self-hosting guide](/docs/deployment/self-hosting#backups) for detailed instructions.

### Can I use AWS S3 instead of MinIO?

Yes. Reqcore supports any S3-compatible storage provider, including AWS S3, DigitalOcean Spaces, Cloudflare R2, and Railway Buckets. Set `S3_FORCE_PATH_STYLE=false` for cloud providers.

## Features

### Does Reqcore have AI-powered features?

Not yet. Resume parsing and AI candidate ranking with transparent matching logic are on the roadmap. When shipped, AI features will follow the "Glass Box" principle — every AI decision will show visible matching logic.

### Can I customize the application form?

Yes. Each job can have custom questions (text, textarea, select, file upload) added to its application form. Applicants see these questions alongside the standard fields when applying through the public job board.

### Does Reqcore support multiple languages?

Yes. The UI supports English, Norwegian, Spanish, French, German, and Vietnamese. Translations are managed via Crowdin. See the [internationalization guide](/docs/contributing/internationalization).

### How does multi-tenancy work?

Each user can belong to multiple organizations. All data (jobs, candidates, applications, documents) is scoped to the active organization. Users can switch organizations in the sidebar. See [authentication](/docs/configuration/authentication).

## Technical

### What database does Reqcore use?

PostgreSQL 16 with Drizzle ORM. Migrations are auto-applied on server startup.

### How are documents secured?

All document access is server-proxied — files are streamed through the authenticated Nitro server. No presigned S3 URLs are exposed to clients. File types are validated using magic bytes, and filenames are sanitized. See [security](/docs/architecture/security).

### Can I extend the API?

Yes. Reqcore uses Nitro server routes. Add new API endpoints in `server/api/` following the existing patterns. See the [API reference](/docs/api/overview).

### How does Reqcore handle rate limiting?

All API endpoints are rate-limited per IP using a sliding window algorithm. Authentication and write endpoints have stricter limits.
