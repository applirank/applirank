# Public Job Board

Reqcore includes a built-in public job board where candidates can browse open positions and submit applications directly.

## How It Works

When a job's status is set to **Open**, it becomes visible on the public job board:

1. **Listing page** at `/jobs` — Shows all open jobs for the organization
2. **Detail page** at `/jobs/[slug]` — Individual job posting with full details
3. **Application form** — Candidates submit name, email, resume, and custom question answers

## URL Structure

| Page | URL | Description |
|------|-----|-------------|
| Job listings | `/jobs` | All open jobs |
| Job detail | `/jobs/senior-software-engineer` | Individual job posting |
| Application | Submit form on job detail page | Form submission |

Slugs are automatically generated from job titles when published. They are URL-friendly and unique within the organization.

## SEO Optimization

Each public job page includes:

### Structured Data (JSON-LD)

Every job detail page includes a `JobPosting` schema with:

- Job title, description, and location
- Employment type (full-time, part-time, etc.)
- Remote work status
- Salary range (if provided)
- Date posted and hiring organization
- Application URL

This helps search engines (Google Jobs, Indeed, etc.) understand and display your job listings in rich results.

### Meta Tags

Full OpenGraph and Twitter Card meta tags are set via `useSeoMeta()`:

- Title, description
- OG image
- Canonical URL

### Sitemap

Open jobs are dynamically included in the sitemap via a server source at `/api/__sitemap__/urls`. When a job is closed or archived, it is automatically removed from the sitemap.

### ISR (Incremental Static Regeneration)

Job pages use ISR with a 1-hour TTL (`isr: 3600`), balancing freshness with performance.

## Application Process

When an applicant submits an application:

1. **Candidate record** is created or reused (deduplication by email)
2. **Application** is linked to the job and candidate
3. **Resume** is uploaded to S3 via the server proxy
4. **Custom question answers** are stored
5. Application appears in the dashboard with **New** status

### Rate Limiting

The public apply endpoint (`POST /api/public/jobs/:slug/apply`) is rate-limited per IP to prevent abuse.

### Validation

- Email format validation
- Required fields enforcement
- File type and size validation (MIME magic bytes)
- Filename sanitization
- Per-candidate document limits (max 20)

## Public API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/public/jobs` | List all open jobs |
| `GET` | `/api/public/jobs/:slug` | Get job details by slug |
| `POST` | `/api/public/jobs/:slug/apply` | Submit an application |

These endpoints require no authentication and only return data for jobs with `open` status.
