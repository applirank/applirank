# Candidate Pipeline

The candidate pipeline is a visual **Kanban board** that helps recruiters track candidates through the hiring process for each job.

## Pipeline Stages

```
new → screening → interview → offer → hired
                                    → rejected
```

| Stage | Description |
|-------|-------------|
| **New** | Application just received. Needs initial review. |
| **Screening** | Reviewing resume, cover letter, and application answers. |
| **Interview** | Candidate is being interviewed. |
| **Offer** | An offer has been extended to the candidate. |
| **Hired** | Candidate accepted the offer and is hired. |
| **Rejected** | Candidate was not selected (can happen at any stage). |

## Using the Pipeline

### Kanban View

Navigate to **Dashboard → Jobs → [Job] → Pipeline** to see the Kanban board.

Each column represents a pipeline stage. Candidate cards show:

- Candidate name and email
- Application date
- Attached documents (resume, cover letter)

### Moving Candidates

Drag and drop candidate cards between columns to update their status. The status change is saved automatically.

### Candidate Table View

Navigate to **Dashboard → Jobs → [Job] → Candidates** for a data table view with:

- Sortable columns
- Status filters
- Candidate detail sidebar

## Candidates

### Organization-Wide Pool

Candidates belong to the organization, not to individual jobs. The same candidate can apply to multiple jobs within your organization.

### Deduplication

Candidates are automatically deduplicated by email address within each organization:

- If someone applies to a second job, their existing candidate record is reused
- All applications are linked to the same candidate profile
- Documents and history are shared across applications

### Candidate Details

Each candidate profile includes:

- Contact information (name, email, phone)
- List of all applications across jobs
- Uploaded documents (resumes, cover letters)
- Application timeline

## Documents

Resumes and cover letters are securely stored in S3-compatible storage:

- Upload during application (public job board) or manually (dashboard)
- All access is authenticated and server-proxied
- PDF preview in a sidebar iframe
- DOC/DOCX files must be downloaded

See [Storage](/docs/configuration/storage) for security details.
