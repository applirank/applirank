# Job Management

Jobs are the core entity in Reqcore. Each job represents an open position within your organization, with a lifecycle from draft to archived.

## Job Lifecycle

```
draft → open → closed → archived
```

| Status | Description |
|--------|-------------|
| **Draft** | Job is being prepared. Not visible publicly. |
| **Open** | Published on the public job board. Accepting applications. |
| **Closed** | No longer accepting applications. Visible in dashboard. |
| **Archived** | Hidden from active job listings. |

## Creating a Job

Navigate to **Dashboard → Jobs → New Job** to create a new job posting.

### Required Fields

| Field | Description |
|-------|-------------|
| Title | Job title (e.g., "Senior Software Engineer") |
| Description | Rich text job description |

### Optional Fields

| Field | Description |
|-------|-------------|
| Location | Where the job is based |
| Employment type | Full-time, part-time, contract, or internship |
| Remote status | On-site, hybrid, or remote |
| Salary range | Minimum and maximum salary with currency |

## Publishing a Job

When you change a job's status from **Draft** to **Open**:

1. A URL-friendly **slug** is automatically generated from the title
2. The job appears on the **public job board** at `/jobs/[slug]`
3. The job is included in the **sitemap** for search engine indexing
4. **Structured data** (JSON-LD `JobPosting`) is added for search engines

## Custom Application Forms

Each job can have custom questions added to its application form:

| Question Type | Description |
|--------------|-------------|
| **Text** | Short text input |
| **Textarea** | Long text input |
| **Select** | Dropdown with predefined options |
| **File** | File upload |

Custom questions are managed in **Dashboard → Jobs → [Job] → Application Form**.

Applicants see these questions alongside the standard fields (name, email, resume) when applying through the public job board.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/jobs` | List all jobs (org-scoped) |
| `POST` | `/api/jobs` | Create a new job |
| `GET` | `/api/jobs/:id` | Get job details |
| `PATCH` | `/api/jobs/:id` | Update a job |
| `DELETE` | `/api/jobs/:id` | Delete/archive a job |
| `GET` | `/api/jobs/:id/questions` | List custom questions |
| `POST` | `/api/jobs/:id/questions` | Add a custom question |

All endpoints require authentication and are scoped to the active organization.
