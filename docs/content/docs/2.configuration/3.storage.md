# Storage

Reqcore stores uploaded documents (resumes, cover letters) in an **S3-compatible** object store. This separates file binary data from the PostgreSQL database.

## Supported Providers

| Provider | Use Case |
|----------|----------|
| **MinIO** | Local development, self-hosted production |
| **AWS S3** | Cloud production |
| **Railway Buckets** | Managed Railway deployment |
| **DigitalOcean Spaces** | Cloud production |
| **Cloudflare R2** | Cloud production |

Any S3-compatible storage provider works.

## Local Development (MinIO)

MinIO is included in the Docker Compose setup. It starts automatically with:

```bash
docker compose up
```

Default configuration in `.env`:

```ini
S3_ENDPOINT=http://localhost:9000
S3_ACCESS_KEY=minioadmin
S3_SECRET_KEY=minioadmin
S3_BUCKET=reqcore
S3_REGION=us-east-1
S3_FORCE_PATH_STYLE=true
```

The MinIO Console is available at [http://localhost:9001](http://localhost:9001).

## Production Configuration

For AWS S3 or other cloud providers, set:

```ini
S3_ENDPOINT=https://s3.amazonaws.com
S3_ACCESS_KEY=AKIAIOSFODNN7EXAMPLE
S3_SECRET_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
S3_BUCKET=my-reqcore-bucket
S3_REGION=eu-west-1
S3_FORCE_PATH_STYLE=false
```

::callout{icon="i-lucide-info" color="info"}
Set `S3_FORCE_PATH_STYLE=true` for MinIO (path-style URLs) and `false` for AWS S3, Railway Buckets, and most cloud providers (virtual-hosted-style URLs).
::

## Security

Reqcore enforces strict document security:

| Measure | Description |
|---------|-------------|
| **Private bucket policy** | MinIO public policies are deleted on every startup |
| **Server-proxied access** | All downloads and previews stream through the authenticated Nitro server |
| **No presigned URLs** | S3 URLs are never exposed to clients |
| **Filename sanitization** | User filenames are sanitized to prevent path traversal and XSS |
| **MIME validation** | File types are validated using magic bytes, not just `Content-Type` headers |
| **Per-candidate limits** | Maximum 20 documents per candidate on public apply endpoints |
| **storageKey hidden** | The internal S3 key is never exposed in API responses |
| **Preview restrictions** | Only PDF files can be previewed inline; DOC/DOCX must be downloaded |
| **Cache headers** | `Cache-Control: private, no-store` on all document endpoints |

## Architecture

```
Client → Authenticated API → Nitro Server → S3/MinIO
                                    ↓
                              Stream bytes
                                    ↓
                              Client receives file
```

The Nitro server acts as a proxy. Clients never communicate directly with S3.

## Bucket Initialization

The `server/plugins/s3-bucket.ts` plugin ensures the S3 bucket exists on startup. For MinIO, it also deletes any public bucket policy to enforce private access.
