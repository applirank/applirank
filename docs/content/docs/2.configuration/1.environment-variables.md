# Environment Variables

All environment variables are validated at startup using Zod in `server/utils/env.ts`. If any required variable is missing or malformed, the server fails immediately with a clear error message.

::callout{icon="i-lucide-shield" color="warning"}
**Security**: No runtime `process.env` access is allowed outside `server/utils/env.ts`. All environment access goes through the validated `env` utility.
::

## Required Variables

These variables must be set for Reqcore to start:

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://user:pass@localhost:5432/reqcore` |
| `BETTER_AUTH_SECRET` | Secret key for session signing | Random 32+ character string |
| `BETTER_AUTH_URL` | Public URL of your Reqcore instance | `https://reqcore.example.com` |

## Storage (S3-Compatible)

Reqcore uses S3-compatible storage for document uploads (resumes, cover letters):

| Variable | Description | Default |
|----------|-------------|---------|
| `S3_ENDPOINT` | S3 API endpoint URL | `http://localhost:9000` |
| `S3_ACCESS_KEY` | S3 access key ID | — |
| `S3_SECRET_KEY` | S3 secret access key | — |
| `S3_BUCKET` | S3 bucket name | `reqcore` |
| `S3_REGION` | S3 region | `us-east-1` |
| `S3_FORCE_PATH_STYLE` | Use path-style URLs (required for MinIO) | `true` |

::callout{icon="i-lucide-info" color="info"}
Set `S3_FORCE_PATH_STYLE=true` for MinIO (local development) and `false` for AWS S3 or Railway Buckets (production).
::

## Site Configuration

| Variable | Description | Default |
|----------|-------------|---------|
| `NUXT_PUBLIC_SITE_URL` | Public URL (used for SEO, sitemap, OpenGraph) | `https://reqcore.com` |

## Demo Mode

| Variable | Description | Default |
|----------|-------------|---------|
| `DEMO_ORG_SLUG` | Organization slug for read-only demo mode | — |
| `LIVE_DEMO_EMAIL` | Pre-filled email on sign-in page | `demo@reqcore.com` |
| `LIVE_DEMO_SECRET` | Pre-filled password on sign-in page | `demo1234` |

## Feedback

| Variable | Description | Default |
|----------|-------------|---------|
| `GITHUB_FEEDBACK_TOKEN` | GitHub personal access token for issue creation | — |
| `GITHUB_FEEDBACK_REPO` | GitHub repository for feedback issues (`owner/repo`) | — |

## Railway-Specific

When deploying on Railway, these variables use Railway's template syntax:

| Variable | Railway Value |
|----------|---------------|
| `DATABASE_URL` | `${{Postgres.DATABASE_URL}}` |
| `S3_ENDPOINT` | `${{Bucket.ENDPOINT}}` |
| `S3_ACCESS_KEY` | `${{Bucket.ACCESS_KEY_ID}}` |
| `S3_SECRET_KEY` | `${{Bucket.SECRET_ACCESS_KEY}}` |
| `S3_BUCKET` | `${{Bucket.BUCKET}}` |
| `S3_REGION` | `${{Bucket.REGION}}` |
| `BETTER_AUTH_URL` | `https://${{RAILWAY_PUBLIC_DOMAIN}}` |

## Generating Secrets

Use the included setup script to generate a `.env` file with random secrets:

```bash
./setup.sh
```

Or generate a secret manually:

```bash
openssl rand -base64 32
```
