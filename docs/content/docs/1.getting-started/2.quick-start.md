# Quick Start

Get Reqcore running locally in under 5 minutes. You need only Docker — no other tools required.

::callout{icon="i-lucide-info" color="info"}
**Windows users:** Open [Git Bash](https://gitforwindows.org) and run all commands there instead of Command Prompt or PowerShell.
::

## Prerequisites

- **Docker** — packages the app, database, and file storage into containers
- **Git** — to clone the repository (optional: you can download a ZIP instead)

| Your OS | How to install Docker |
|---------|----------------------|
| **Mac** | [Download Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/) |
| **Windows** | [Download Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/) |
| **Linux** | Follow the [Docker Engine install guide](https://docs.docker.com/engine/install/) |

Verify Docker is running:

```bash
docker --version
# Docker version 27.x.x
```

## Step 1 — Clone the Repository

```bash
git clone https://github.com/reqcore-inc/reqcore.git
cd reqcore
```

::callout{icon="i-lucide-download" color="neutral"}
Don't have `git`? [Download the ZIP](https://github.com/reqcore-inc/reqcore/archive/refs/heads/main.zip) and unzip it manually.
::

## Step 2 — Generate Secrets

This creates a `.env` file containing random passwords and secrets. Run this once:

```bash
./setup.sh
```

You'll see: `✅ .env generated with random secrets.`

::callout{icon="i-lucide-terminal" color="warning"}
**Windows CMD / PowerShell?** Run `cp .env.example .env` instead, then open `.env` and replace every placeholder value with a random string.
::

## Step 3 — Start the App

```bash
docker compose up
```

**The first run takes 3–5 minutes** while Docker builds the app image and downloads dependencies. Subsequent starts take seconds.

When you see:

```
app  | Listening on http://[::]:3000
```

The app is ready.

## Step 4 — Open Reqcore

Go to **[http://localhost:3000](http://localhost:3000)** in your browser.

Click **Sign up** to create your account and first organization.

## Load Demo Data (Optional)

Want to explore with pre-filled jobs, candidates, and a pipeline? Open a **new terminal** while the app is running:

```bash
docker compose exec app npm run db:seed
```

Then sign in with:

| Field | Value |
|-------|-------|
| Email | `demo@reqcore.com` |
| Password | `demo1234` |

## What's Running

| Service | URL | Description |
|---------|-----|-------------|
| **App** | [localhost:3000](http://localhost:3000) | The Reqcore web UI |
| **MinIO Console** | [localhost:9001](http://localhost:9001) | File storage browser (S3-compatible) |
| **Adminer** | [localhost:8080](http://localhost:8080) | Database browser (with `--profile tools`) |

To enable Adminer:

```bash
docker compose --profile tools up
```

## Managing Your Instance

```bash
# Stop the app (data is kept)
docker compose down

# Start it again
docker compose up

# Rebuild after pulling new code
docker compose up --build

# Stop and delete ALL data (irreversible)
docker compose down -v
```

## Updating to a New Release

```bash
# 1. Pull the latest code
git pull origin main

# 2. Rebuild and restart
docker compose up --build -d

# 3. Verify it's running
docker compose logs app --tail 20
```

Look for `Listening on http://[::]:3000`. Database migrations are applied automatically on startup.
