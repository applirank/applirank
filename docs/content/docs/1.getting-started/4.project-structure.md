# Project Structure

Reqcore follows the **Nuxt 4** directory structure with application source code inside `app/` and server code at the project root.

## Directory Layout

```
reqcore/
├── app/                          # Client source (Nuxt 4 srcDir)
│   ├── app.vue                   # Root component
│   ├── assets/
│   │   └── css/main.css          # Tailwind CSS entry point
│   ├── components/               # Auto-imported Vue components
│   ├── composables/              # Auto-imported composables (useXxx)
│   ├── layouts/                  # Layout components
│   │   ├── dashboard.vue         # Sidebar + main area
│   │   ├── auth.vue              # Centered card for sign-in/sign-up
│   │   └── public.vue            # Header/footer for public pages
│   ├── middleware/                # Client-side route middleware
│   ├── pages/                    # File-based routing
│   ├── plugins/                  # Client-side Nuxt plugins
│   └── utils/                    # Auto-imported utilities
├── server/                       # Nitro server (at project root)
│   ├── api/                      # API routes (/api/*)
│   ├── database/
│   │   ├── schema/               # Drizzle ORM table definitions
│   │   └── migrations/           # Generated SQL migrations
│   ├── middleware/                # Global server middleware
│   ├── plugins/                  # Server startup plugins
│   └── utils/                    # Auto-imported server utilities
├── content/                      # Markdown content (@nuxt/content v3)
│   ├── blog/                     # Blog articles
│   └── catalog/                  # Feature catalog
├── docs/                         # Documentation site (Docus — you are here)
├── public/                       # Static assets
├── shared/                       # Code shared between app and server
├── i18n/                         # Translation files
├── e2e/                          # Playwright end-to-end tests
├── docker-compose.yml            # Postgres + MinIO + Adminer
├── Dockerfile                    # Multi-stage production build
├── drizzle.config.ts             # Drizzle Kit configuration
├── content.config.ts             # Nuxt Content collection definitions
├── nuxt.config.ts                # Nuxt configuration
└── package.json                  # Dependencies
```

## Key Directories

### `app/` — Client Application

All frontend Vue code lives here. This is the Nuxt 4 `srcDir`:

- **`pages/`** — File-based routing. Each `.vue` file becomes a route.
- **`components/`** — Auto-imported Vue components. No manual imports needed.
- **`composables/`** — Reusable reactive logic (e.g., `useJobs()`, `useCandidates()`).
- **`layouts/`** — Page layouts (dashboard with sidebar, auth card, public).
- **`middleware/`** — Client-side route guards (auth, guest, require-org).

### `server/` — Backend

Nitro-powered server code at the project root:

- **`api/`** — REST API routes, organized by domain (jobs, candidates, documents).
- **`database/schema/`** — Drizzle ORM table definitions.
- **`database/migrations/`** — SQL migration files (auto-generated by Drizzle Kit).
- **`utils/`** — Auto-imported utilities (db, auth, env, s3, rate limiter).
- **`plugins/`** — Server startup plugins (auto-migrate, S3 bucket setup).

### `content/` — Blog & Content

Markdown files powered by `@nuxt/content` v3:

- **`blog/`** — Blog articles with YAML frontmatter.
- **`catalog/`** — Feature catalog entries.

### `public/` — Static Assets

Files served at the root URL, not processed by the build pipeline. Favicons, images, and static files.

## Key Files

| File | Purpose |
|------|---------|
| `nuxt.config.ts` | Nuxt configuration (modules, site config, route rules) |
| `content.config.ts` | Nuxt Content collection schemas |
| `drizzle.config.ts` | Drizzle Kit database configuration |
| `docker-compose.yml` | Local development services (Postgres, MinIO, Adminer) |
| `Dockerfile` | Multi-stage production Docker image |
| `setup.sh` | One-time secret generator (creates `.env`) |

## The `~` Alias

In Nuxt 4, the `~` and `@` aliases point to `app/`:

```ts
// This resolves to app/components/AppSidebar.vue
import AppSidebar from '~/components/AppSidebar.vue'
```

## Auto-Imports

Nuxt 4 automatically imports from:

| Directory | Available In |
|-----------|-------------|
| `app/components/` | Vue templates |
| `app/composables/` | `<script setup>` and composables |
| `app/utils/` | `<script setup>` and composables |
| `server/utils/` | Server routes and middleware |
| `shared/` | Both app and server |
